# Business & Pricing

**Last updated:** 2026-05-05

---

## Commercial Model

The product should start open-source and self-serve.

No calls, no audits, no enterprise sales motion in v1.

The free product creates adoption. Paid surfaces add convenience, history, collaboration, and packaged templates.

---

## Pricing

| Tier | Price | Includes |
|---|---:|---|
| Open Source | Free | CLI, GitHub Action, local reports, starter checks |
| Pro | €19-29/month | hosted run history, private projects, trend charts |
| Team | €99-199/month | more runs, shared projects, Slack alerts, team controls |
| Template Packs | €29-99 one-time | focused packs for support, JSON extraction, tool agents |
| Self-host Team License | €499-1,999/year | later, only if inbound demand appears |

Do not start with enterprise pricing.

---

## Distribution

Primary channels:

- GitHub
- npm
- GitHub Marketplace
- Hacker News launch
- Reddit developer communities
- SEO docs for "LLM CI", "prompt regression testing", and "JSON schema LLM tests"

The product needs a strong example repo. Developers should understand the value from one pull request screenshot.

---

## Conversion Path

1. Developer finds GitHub repo or article.
2. Installs CLI.
3. Adds GitHub Action.
4. Gets useful PR report.
5. Wants hosted history or team visibility.
6. Converts to Pro or Team.

---

## Revenue Goals

### First 90 Days

- 500 GitHub stars or equivalent developer attention
- 50 real installs
- 10 recurring users
- first 5 paid Pro users

### First 6 Months

- 100+ active repos
- 25+ paid accounts
- €1k+ MRR
- clear signal on whether hosted history or template packs convert better

---

## Crowding Assessment

The broad market is crowded.

Avoid direct competition with:

- Braintrust
- LangSmith
- Langfuse
- Galileo
- Promptfoo
- DeepEval

The narrow wedge is still plausible:

- CI-native
- tiny setup
- excellent PR reports
- structured output checks first
- no dashboard required

---

## Kill Criteria

Stop or pivot if, after a real launch and repeated distribution attempts:

- developers do not commit configs to repos
- GitHub Action installs are weak
- users run it once but do not repeat
- no one asks for hosted history, team reports, or templates

Do not keep adding features to compensate for weak activation.
